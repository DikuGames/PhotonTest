# PhotonTest

# Проект построен вокруг общего игрового цикла:

- подключение из лобби
- вход в комнату
- подготовка игровой сцены
- совместный сбор артефактов
- завершение матча и возврат в лобби

#  Общая структура

Код разделен по зонам ответственности.

#  `Gameplay`

Содержит игровую и доменную логику:

- `Artifacts` - артефакты, реестр объектов, подсчет прогресса
- `Player` - контроллер игрока
- `Camera` - создание локальной камеры игрока
- `Match` - логика завершения матча
- `StateMachine` - игровой цикл сцены `Game`
- `EntryPoint` - сценовые зависимости и точки входа
- `Input` - абстракция ввода
- `StaticData` - загрузка конфигов

# `Networking`

# Содержит интеграцию с Photon:
- `Lobby` - подключение к Photon Cloud и работа с комнатами
- `Artifacts` - синхронизация сбора артефактов и room state
- `Room` - выход из комнаты и возврат в `Lobby`
- `Connection` - обработка потери соединения
- `EventCodes` - коды сетевых событий

# Game Loop

Игровая сцена построена через state machine:
- `PrepareGameState`
- `GameState`
- `GameOverState`

# `PrepareGameState`
Подготавливает сцену к началу матча:
- ожидает готовности Photon room
- применяет начальное сетевое состояние
- спавнит локального игрока
- переводит игру в основное состояние

#  `GameState`
Основное игровое состояние. В нем выполняется обычный игровой цикл до завершения матча.

#  `GameOverState`
Отвечает за завершение матча:
- выход из комнаты
- возврат в сцену `Lobby`

# Состав player-логики
- `PlayerController` - основная точка сборки player-компонентов
- `PlayerMovement` - перемещение через `CharacterController`
- `PlayerLook` - поворот тела и камеры
- `PlayerInteraction` - взаимодействие с collectable-объектами
- `PlayerDetector` - визуальное и логическое состояние детектора

# Артефакты и общий прогресс
Артефакты заранее размещены на сцене и регистрируются в `ArtifactRegistry`.

# Поток сбора
- локальный игрок инициирует запрос на сбор
- запрос отправляется мастеру комнаты
- мастер подтверждает сбор
- состояние комнаты обновляется через `Room Custom Properties`
- все клиенты применяют обновленное состояние локально

Такой подход позволяет:
- синхронно убирать артефакты у всех игроков
- поддерживать общий счетчик прогресса
- корректно восстанавливать состояние при позднем входе в комнату

`ArtifactCollectionService` отслеживает общий прогресс и уведомляет остальные системы о достижении финального состояния.

# Сетевое поведение
#  Подключение и комнаты
- подключение выполняется через `PhotonLobbyNetwork`
- если доступна существующая комната, используется она
- иначе создается новая

#  Используемые технологии
- `Unity 6`
- `PUN 2`
- `Photon Cloud`
- `Zenject`

# `PlayerConfig`
В конфиге игрока настраиваются:
- префаб игрока
- префаб локальной камеры
- параметры движения и взгляда
- дистанция взаимодействия
- цвета состояний детектора

# \## Управление
- `WASD` - движение
- `Mouse` - взгляд
- `LMB` - переключение детектора
- `E` - взаимодействие с артефактами
